# v1.1.0 版本已发布：新增文件管理器，支持上传下载文件到远程服务器

SSH Config Manager v1.1.0 现已发布。本次更新为你带来了**内置的文件管理器**，让你可以直接在应用中浏览、上传、下载和管理远程文件，无需在终端标签页之间来回切换，也无需打开独立的 SFTP 客户端。

## 新功能

### 文件管理器 (SFTP)

文件管理器直接集成在你现有的 SSH 连接中。连接到远程主机后，你现在可以通过可视化界面访问文件系统。

- **上传和下载**文件和文件夹，支持拖拽上传，也可以点击浏览本地文件
- **重命名和删除**远程服务器上的文件或文件夹
- **创建目录**，方便组织远程文件结构
- **文件元数据展示**，显示所有者、用户组、权限、修改日期和文件大小
- **一键切换隐藏文件显示**，方便管理 `.bashrc`、`.env` 等配置文件

### 其他新功能

- **自动关闭旧浏览器标签页**：仅保留最新的标签页，避免频繁重连时标签堆积
- **复制 SSH 连接**：在当前分组中快速复制已有的 SSH 连接，修改后即可创建新连接，无需从头填写
- **打开本地终端**：直接从应用中启动系统终端，方便快速执行本地命令

## 修复

- **修复含 `RemoteCommand` 的主机的 SSH 连接测试**：当远程主机配置了 `RemoteCommand` 指令时，连接测试不再挂起或失败
- **终端选择器宽度**：下拉菜单在窄屏上不再溢出或被截断

## 更新与打磨

- **页脚**：现已包含隐私政策、使用条款、联系方式和 GitHub 仓库链接
- **Favicon**：统一全站图标，品牌展示更一致
- **首页**：更新设计与内容布局

## 为什么这很重要

在 v1.1.0 之前，你可以管理 SSH 连接并启动终端，但要传输文件，还得打开独立的 SFTP 工具或手动执行 `scp` 命令。现在你可以在一个地方完成所有操作，对于频繁调整配置文件或向远程服务器部署小资源时尤其方便。

本地终端启动和连接复制功能进一步简化了你的工作流，让 SSH Config Manager 成为更完整的远程服务器日常管理工具。
